# Dataroom Schema — `dataroom/`

Raw working material. Both founder-shared and our own production live here.

## Subfolders

| Folder | Contents |
|---|---|
| `deck/` | Founder deck + iterations (PDF, .pptx, link to live deck) |
| `interviews/` | Notes from founder, customer, and reference calls. One file per call: `YYYY-MM-DD-<name>-<role>.md` |
| `market/` | Market sizing models, industry reports, analyst notes |
| `product/` | Demo notes, product observations, technical DD |
| `financials/` | Founder model, historicals, our analysis |
| `legal/` | Cap table, term sheet drafts, SAFE/SHA docs, IP filings |
| `references/` | Backchannel notes — investor/operator references on the team |

## Conventions

- File names: kebab-case, date-prefixed where chronology matters
- Every file states its source at the top: `Source: <founder / public / our analysis>` + date
- `INDEX.md` lists what's in the dataroom — update on every add
- Sensitive material (cap table, term sheet) flagged at the top of the file

## What does *not* go here

- Synthesized analysis → `research.md`, `competition.md`, `risks.md`
- Memo content → `memo.md`
- Decision artifacts → `recommendation.md`, `review-log.md`
