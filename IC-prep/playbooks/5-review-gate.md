# Playbook 5 — Review Gate

**When to use:** Memo, deck outline, and appendix all drafted. Quality check before IC.

## Steps

1. **Run the rubric** — `_config/review-rubric.md` "After stage 4" checklist
2. **For each failure:**
   - Identify the root cause (evidence gap / analysis gap / contradiction / mandate mismatch)
   - Determine the reroute stage (2, 3, or within 4)
   - Log in `review-log.md`: date, issue, reroute, target close
3. **Internal review with sponsoring partner** — they sign off in `review-log.md`
4. **If all pass** → proceed to `6-final-package.md`

## Reroute matrix

| Failure | Reroute to |
|---|---|
| Missing dataroom item | 2 |
| Unsupported claim | 2 (gather) or 3 (cite existing) |
| Weak market sizing | 3 |
| Memo/deck contradiction | within 4 |
| Unresolved portfolio conflict | 3 (re-check) |
| Mandate mismatch | escalate; do not proceed |

## Hard blockers — never pass

See `_config/review-rubric.md` "Hard blockers" section.

## Exit criterion

All rubric checks pass; sponsoring partner signed off in `review-log.md`.
